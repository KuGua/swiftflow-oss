# SwiftFlow OSS

[English](./README.md)

SwiftFlow OSS 是一个个人项目，核心想法是尝试用软件系统把线下服务业中原本分散、依赖人工整理的运营流程，变得更结构化、更可追踪，也更容易扩展。

这个项目的起点，来自一个很直接的观察：在不少线下零售和服务场景里，排班与人员信息仍然高度依赖聊天沟通、表格整理和人工跟进。这样的方式虽然能运转，但效率不高，也很容易在信息记录、规则传递和排版整理过程中出现遗漏、不一致或重复确认的问题。

SwiftFlow OSS 想解决的，不只是“生成班表”这一个动作，而是把门店配置、员工准备度、可排时间、规则约束和排班执行这些原本分散的环节，放进一个更完整的业务工作流里。

## 为什么做这个项目

在线下服务团队中，排班本质上不只是一个时间表问题，更是一个信息组织与流程规范问题。

常见痛点包括：

- 员工可上班时间分散在聊天记录、临时备注或口头沟通中
- 人力需求和排班规则缺乏结构化沉淀，往往依赖经验判断
- 不同门店之间的差异需要管理者手动记忆和处理
- 班表调整后的变更不容易追溯，也不容易形成统一流程
- 关键运营知识更多依赖个人经验，而不是系统化配置

因此，这个项目的目标不是只做一个“排班页面”，而是尝试构建一个更像真实业务系统的排班基础平台。

## 项目定位

SwiftFlow OSS 是一个面向多门店零售与服务业场景的开源排班系统，重点覆盖排班前的配置准备、排班中的生成与调整，以及排班后的查看与协同。

目前包含的核心能力有：

- 面向多门店场景的统一排班工作台，支持周视图与日视图编辑
- 门店级营业时间、人力需求与排班规则配置
- 员工档案、技能矩阵、钥匙权限与双周可排时间管理
- 管理员、区域经理、店长、员工等多角色权限分层
- 排班异常识别、修复与批量生成能力
- 本地开发支持 SQLite，生产环境推荐 PostgreSQL

## 项目价值

从产品和业务流程角度看，SwiftFlow OSS 主要想提升三件事：

- 信息清晰度：把员工能力、可排时间、门店规则等信息从零散沟通中抽离出来，形成结构化数据
- 流程一致性：让不同门店在共享框架下运转，同时保留门店级配置差异
- 运营效率：让管理者可以在一个工作台内完成生成、检查和调整，而不是在多个工具之间切换

## 角色设计

当前项目将平台管理和日常排班运营做了角色区分：

- `admin`：负责用户、角色以及平台级门店基础数据管理
- `area_manager`：负责所分配区域内的门店排班运营与协调
- `store_manager`：负责所属门店的排班与人员配置维护
- `staff`：查看个人班表并维护个人自助信息

其中 `area_manager` 是比较关键的业务角色，适合管理跨门店、跨区域的人力与排班工作。该角色在被管理员分配区域范围后，可以使用区域范围内的员工管理、规则配置和排班生成等核心功能。

## 关键业务流程

项目目前聚焦于门店人力运营中的核心环节：

- 管理端工作台与运营总览
- 门店目录、区域归属、营业时间、人力需求与排班规则维护
- 员工资料、技能、授权门店、关系约束与可排时间编辑
- 排班生成、周历调整、日时段检查与手动修正
- 员工端个人班表与个人资料查看

## 技术栈

- 前端：Vue 3 + Vite
- 后端：FastAPI
- 数据层：SQLAlchemy
- 数据库：本地开发使用 SQLite，生产部署推荐 PostgreSQL

## 仓库结构

- `SwiftFlow-Frontend/`：前端应用
- `SwiftFlow-Backend/`：后端 API、数据库与排班逻辑
- `algorithms/`：排班算法参考
- `docs/`：部署说明与发布材料

## 快速开始

### 前端

```bash
cd SwiftFlow-Frontend
npm install
npm run dev
```

### 后端

```bash
cd SwiftFlow-Backend
pip install -r requirements.txt
python scripts/init_database.py
uvicorn app:app --reload --host 127.0.0.1 --port 8000
```

初始化脚本会创建数据库结构，并为本地环境准备默认的 `admin / admin` 账户。

同时也会创建默认的 `area_manager / area_manager` 账户，方便直接测试区域经理工作流。管理员登录后，可在用户管理中为该账号分配具体区域范围。

## 生产部署说明

- 本地开发默认使用 SQLite
- 生产环境建议切换到 PostgreSQL
- 具体部署方式可参考 `SwiftFlow-Backend/docs/postgresql_deployment.md`
